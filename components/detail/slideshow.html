Caveats

In order to use the slideshow you must provide a height for your slides. This can be done either by specifying a height on the .slide div itself, using the responsive height CSS trick (see the examples here or below), or by simply putting content in the div that defines its own height (e.g. text or an img element).

Responsive height CSS trick

Sometimes you want to use a single div as your slide, with a background image, and background-size set to cover. Doing so, you will have to hardcode the height of the image, as the browser will not automatically size your div for you like it does with an img element. So, in order to maintain a given element’s aspect ratio (e.g. 4:3, 16:9, etc.) without having to set a specific height on it, we can use a little CSS trick:

.slide {
    height: 0;
    padding-bottom: 66.667%;
}

Here we set the actual div‘s heigh to 0 so that it does not take up any space of its own, instead setting its padding-bottom to a percentage value that represents the desired ratio of height to width of the element.

In this case we use 66.67% which is the aspect ratio for 3:2, which is common for photographs. To calculate this value simply take the height, divide it by the width, and then turn that fraction into a percentage:

2 / 3 = 0.6667
0.6667 * 100% = 66.67%

Common ratios you may wish to use:

  • 1:1: 100% (a.k.a. square)
  • 2:1: 50%
  • 3:2: 66.67%
  • 4:3: 75%
  • 16:10: 62.5%
  • 16:9: 56.25%
